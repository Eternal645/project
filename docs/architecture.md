# Архитектура

## Обзор

Проект разделен на несколько частей:

- `main.py` - точка входа desktop-приложения и главное окно.
- `app/config.py` - настройки приложения: путь к базе, тема интерфейса, меню и карточки главного экрана.
- `app/database.py` - подключение к SQLite, создание схемы, загрузка CSV и получение именованных SQL-запросов.
- `app/ui_tables.py` - универсальные CRUD-окна для таблиц.
- `app/ui_order_form.py` - форма заказа и операции со складом/доставкой.
- `app/ui_reports.py` - аналитические и параметрические отчеты.
- `packages/core/` - переиспользуемая логика расчета и проверки строк заказа.
- `sql/` - схема и SQL-запросы.
- `data/` - начальные CSV-данные.
- `tests/unit/` - unit-тесты переиспользуемого ядра.
- `tests/integration/` - интеграционные тесты базы, CSV и именованных SQL-запросов.
- `infra/compose.yaml` - контейнерная проверка проекта.
- `scripts/` - короткие команды запуска приложения, тестов и проверки документации.
- `pyproject.toml` - описание Python-пакета для локальной установки и публикации reusable-компонента.

## Границы ответственности

GUI отвечает за ввод пользователя, отображение таблиц и вызов операций. База данных хранит ограничения целостности: внешние ключи, неотрицательные цены и остатки, положительные количества. Пакет `packages/core` содержит чистые функции, которые можно тестировать без запуска Tkinter.

## Решения

- SQLite выбран как локальная БД без отдельного сервера.
- CSV используются как воспроизводимые начальные данные.
- SQL-запросы таблиц, справочников, отчётов и параметрического поиска вынесены из Python-кода в `sql/queries.sql`.
- Путь к базе задаётся через `APP_DB_PATH` с безопасным значением по умолчанию `db/mebel.db`.
- Статусы управляются флагами в справочниках, чтобы логика не зависела от конкретных текстовых названий.
- Тесты используют in-memory SQLite, поэтому не портят рабочий файл `db/mebel.db`.
- Переиспользуемый компонент можно установить локально командой `make install-lib-local`; публикация в TestPyPI подготовлена командой `make publish-lib-testpypi`.
- Опубликованная версия компонента доступна в TestPyPI: `https://test.pypi.org/project/furniture-store-core/0.1.0/`.

## Риски и развитие

- Часть операций формы заказа все еще находится в GUI-модуле. Для роста проекта ее можно дальше переносить в сервисный слой.
- Tkinter-тесты не добавлены, потому что для учебной проверки важнее стабильность данных, SQL и бизнес-правил.
- Центральная публикация требует учетной записи TestPyPI и токена доступа; опубликованная версия доступна публично, но сам секрет не хранится в репозитории.
