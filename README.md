# Отчёт по проекту: БД «Мебельный магазин»

## 1. Целевая аудитория и задачи

**Пользователи:** менеджеры продаж, кладовщик, бухгалтер, директор.

| Задача | Таблица(ы) |
|---|---|
| Вести каталог мебели по категориям | Товары, Категории |
| Оформлять и отслеживать заказы | Заказы, СтрокиЗаказа |
| Контролировать складские остатки | Товары (поле остаток), СтрокиПоставки |
| Вести базу клиентов и сотрудников | Клиенты, Сотрудники |
| Учитывать поставки и поставщиков | Поставщики, Поставки, СтрокиПоставки |
| Контролировать доставку заказов | Доставки |
| Формировать аналитические отчёты | все таблицы |

## 3. Описание процесса в нотации IDEF0

### Контекстная диаграмма A-0

| Роль | Объекты (соответствие таблицам БД) |
|---|---|
| **Вход** | Клиенты → Клиенты; Товары → Товары; Поставщики → Поставщики |
| **Управление** | Ценовая политика (поле цена в Товарах); Законодательство |
| **Механизм** | Сотрудники → Сотрудники; ИС → файл mebel.db |
| **Выход** | Заказы → Заказы; Доставки → Доставки; Поставки → Поставки; Отчёты |

### Декомпозиция A0

| Блок | Процесс | Таблицы |
|---|---|---|
| A1 | Управление каталогом | Категории, Товары |
| A2 | Обработка заказов | Заказы, СтрокиЗаказа |
| A3 | Управление складом | Товары (остаток), СтрокиПоставки |
| A4 | Работа с клиентами | Клиенты, Доставки |
| A5 | Снабжение | Поставщики, Поставки, СтрокиПоставки |
| A6 | Аналитика | все таблицы (8 отчётов) |

## 4. Схема данных (10 таблиц)

```
Категории(id PK, название, описание)
    ↓ 1:N
Товары(id PK, id_категории FK, наименование, производитель, цена, остаток, описание)
    ↓ 1:N                        ↓ 1:N
СтрокиЗаказа(id PK,          СтрокиПоставки(id PK,
  id_заказа FK,                 id_поставки FK,
  id_товара FK,                 id_товара FK,
  количество,                   количество,
  цена_единицы)                 закупочная_цена)
    ↑ N:1                            ↑ N:1
Заказы(id PK,                 Поставки(id PK,
  id_клиента FK,                id_поставщика FK,
  id_сотрудника FK,             дата_поставки,
  дата_заказа,                  сумма)
  статус,                            ↑ N:1
  итого,                    Поставщики(id PK, название,
  способ_оплаты)              контактное_лицо, телефон, email)
    ↓ 1:1
Доставки(id PK, id_заказа FK,
  адрес_доставки, дата_доставки, курьер, статус)

Клиенты(id PK, фио, телефон, email, адрес, дата_регистрации)
    ↑ N:1 (в Заказы)

Сотрудники(id PK, фио, должность, телефон, зарплата)
    ↑ N:1 (в Заказы)
```

## 6. SQL-запросы (реализованы в ui_reports.py)

```sql
-- 1. Выручка по месяцам
SELECT substr(дата_заказа,1,7) AS период,
       COUNT(*) AS заказов,
       printf('%.2f', SUM(итого)) AS выручка
FROM Заказы WHERE статус = 'выполнен'
GROUP BY substr(дата_заказа,1,7) ORDER BY период;

-- 2. ТОП-10 товаров по продажам
SELECT т.наименование,
       SUM(сз.количество) AS продано_шт,
       printf('%.2f', SUM(сз.количество * сз.цена_единицы)) AS выручка
FROM СтрокиЗаказа сз
JOIN Товары т ON сз.id_товара = т.id
JOIN Заказы з ON сз.id_заказа = з.id
WHERE з.статус != 'отменён'
GROUP BY т.id ORDER BY продано_шт DESC LIMIT 10;

-- 3. VIP-клиенты (сумма > 50 000)
SELECT к.фио, к.телефон, COUNT(з.id) AS заказов,
       printf('%.2f', SUM(з.итого)) AS сумма
FROM Клиенты к JOIN Заказы з ON к.id = з.id_клиента
WHERE з.статус != 'отменён'
GROUP BY к.id HAVING SUM(з.итого) > 50000
ORDER BY SUM(з.итого) DESC;

-- 4. Остатки на складе
SELECT к.название AS категория, т.наименование,
       т.остаток, printf('%.2f', т.цена) AS цена,
       printf('%.2f', т.остаток * т.цена) AS стоимость_запаса
FROM Товары т JOIN Категории к ON т.id_категории = к.id
ORDER BY к.название, т.наименование;

-- 5. Критически малый остаток (< 4 шт.)
SELECT к.название, т.наименование, т.остаток
FROM Товары т JOIN Категории к ON т.id_категории = к.id
WHERE т.остаток < 4 ORDER BY т.остаток;

-- 6. Продажи по сотрудникам
SELECT COALESCE(с.фио,'(не указан)') AS сотрудник,
       с.должность, COUNT(з.id) AS заказов,
       printf('%.2f', SUM(з.итого)) AS выручка
FROM Заказы з LEFT JOIN Сотрудники с ON з.id_сотрудника = с.id
WHERE з.статус != 'отменён'
GROUP BY з.id_сотрудника ORDER BY SUM(з.итого) DESC;

-- 7. Статусы доставок
SELECT д.статус, COUNT(*) AS кол_во,
       GROUP_CONCAT(д.id_заказа, ', ') AS номера_заказов
FROM Доставки д GROUP BY д.статус;

-- 8. Итоги по поставщикам
SELECT пост.название, COUNT(п.id) AS поставок,
       printf('%.2f', SUM(п.сумма)) AS итого
FROM Поставки п JOIN Поставщики пост ON п.id_поставщика = пост.id
GROUP BY пост.id ORDER BY SUM(п.сумма) DESC;
```

## 7. Интерфейс (GUI)

**Главное окно** содержит 4 раздела:
- **Каталог и склад** - Товары, Категории, Поставщики, Поставки
- **Продажи** - Заказы, Строки заказа, Доставки
- **Контрагенты и персонал** - Клиенты, Сотрудники
- **Аналитика** - 8 отчётов

Каждое окно таблицы поддерживает: просмотр, добавление, редактирование, удаление, **живой поиск**, **сортировку** по любой колонке, выпадающие списки для FK-полей.
