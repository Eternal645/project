# БД «Мебельный магазин» - Python + SQLite + Tkinter

Учебный проект: desktop-приложение для автоматизации мебельного магазина.  
Стек: Python 3.11, SQLite 3, Tkinter (GUI), CSV (начальные данные).

---

## Быстрая проверка

```bash
python -m pip install -r requirements.txt
python -m unittest discover -s tests
python main.py
```

Если установлен `make`, те же действия доступны через:

```bash
make setup
make test
make run
```

Полная локальная проверка:

```bash
make check
```

---

## Запуск приложения

```bash
python main.py
```

База данных `db/mebel.db` создаётся автоматически при первом запуске.  
Если схема БД изменилась — удалите `db/mebel.db` и запустите снова.

Для локальной конфигурации можно ориентироваться на `.env.example`.

---

## Тесты

Автоматические тесты находятся в `tests/`.

Проверяется:
- создание SQLite-базы из `sql/schema.sql`
- загрузка начальных CSV-данных
- доступность именованных SQL-запросов
- ограничения схемы БД
- переиспользуемая логика расчёта заказа из `packages/core`

Запуск:

```bash
python -m unittest discover -s tests
```

Тесты используют in-memory SQLite и не изменяют рабочий файл `db/mebel.db`.

## Схема базы данных (14 таблиц)

### Справочники (lookup-таблицы)

| Таблица | Поля | Назначение |
|---|---|---|
| СтатусыЗаказа | id, название, is_final, is_delivering, is_cancelled | Статусы жизненного цикла заказа |
| СпособыОплаты | id, название | Наличные, карта, онлайн |
| СтатусыДоставки | id, название, is_transit, is_delivered | Статусы доставки |
| МестаХранения | id, название, описание | Секции склада |

### Основные справочники

| Таблица | Ключевые поля |
|---|---|
| Категории | id, название, описание |
| Поставщики | id, название, контактное_лицо, телефон, email |
| Сотрудники | id, фио, должность, телефон, зарплата |
| Клиенты | id, фио, телефон, email, адрес, дата_регистрации |
| Товары | id, id_категории FK, наименование, производитель, цена, остаток, id_места_хранения FK |

### Транзакционные таблицы

| Таблица | Ключевые поля |
|---|---|
| Заказы | id, id_клиента FK, id_сотрудника FK, дата_заказа, id_статуса FK, итого, id_способа_оплаты FK |

## Бизнес-логика

Вся логика сосредоточена в `ui_order_form.py` и `ui_tables.py`. Справочные значения хранятся в БД и загружаются через флаги - никакого хардкода строк в коде.

### Жизненный цикл заказа

```
новый → в обработке → доставляется → выполнен
                    ↘ отменён
```

| Переход статуса | Автоматическое действие |
|---|---|
| любой → **доставляется** | Товары списываются со склада (`остаток - количество`) |
| любой → **доставляется** | Автоматически создаётся запись в таблице Доставки |
| **доставляется** → **отменён** | Товары возвращаются на склад |
| **доставляется** → **выполнен** | Склад не меняется (товар уже списан) |

### Автосоздание доставки

При смене статуса заказа на «доставляется» запись в Доставки заполняется автоматически:

| Поле | Откуда берётся |
|---|---|
| id_заказа | Текущий заказ |
| адрес_доставки | Адрес клиента из таблицы Клиенты |
| дата_доставки | Дата заказа |
| курьер | Случайный сотрудник из таблицы Сотрудники |
| статус | «запланирована» (первый статус доставки) |

### Статусы доставки

| Статус | Флаг | Действие |
|---|---|---|
| запланирована | - | Ничего |
| в пути | is_transit=1 | - (информационный) |
| доставлена | is_delivered=1 | Заказ автоматически переходит в статус «выполнен» |

## Архитектурные решения

### Запускаемая часть и переиспользуемая логика

`main.py` и GUI-модули в `app/ui_*.py` отвечают за запуск окна, ввод пользователя и отображение данных.  
Переиспользуемая логика расчёта заказа вынесена в `packages/core`; её можно проверять без запуска Tkinter и без подключения SQLite.

### Данные вынесены в CSV
Все начальные данные (статусы, товары, клиенты и т.д.) хранятся в `data/*.csv` и загружаются в БД при первом запуске. Чтобы изменить статусы или добавить новые — достаточно отредактировать CSV, код трогать не нужно.

### SQL вынесен в queries.sql
Запросы таблиц, справочников, отчётов, прибыли и параметрического поиска хранятся в `sql/queries.sql` с именованными метками:
```sql
-- [report_выручка_по_месяцам]
SELECT ...
```
В коде запросы вызываются по имени: `get_query("report_выручка_по_месяцам")`.

### Настройки вынесены в config.py
Путь к базе, версия приложения, размер окна, цвета, пункты меню и карточки главного экрана находятся в `app/config.py`. Путь к базе можно переопределить через переменную окружения `APP_DB_PATH`.

### Флаги вместо строк
Бизнес-логика привязана к флагам в таблицах-справочниках, а не к названиям статусов:

| Таблица | Флаг | Смысл |
|---|---|---|
| СтатусыЗаказа | is_delivering | Заказ в доставке |
| СтатусыЗаказа | is_cancelled | Заказ отменён |
| СтатусыЗаказа | is_final | Заказ выполнен |
| СтатусыДоставки | is_delivered | Доставка завершена |

## Отчёты

Все отчёты доступны через вкладку **Аналитика** в главном окне.

### Готовые отчёты
| Отчёт | Описание |
|---|---|
| Выручка по месяцам | Количество заказов и сумма по каждому месяцу |
| ТОП-10 товаров | Самые продаваемые позиции по количеству и выручке |
| Местонахождение товаров | Где на складе лежит каждый товар |
| VIP-клиенты | Клиенты с суммой заказов > 50 000 ₽ |
| Остатки на складе | Текущие остатки и стоимость запаса |
| Малый остаток | Товары с остатком менее 4 штук |
| Продажи по сотрудникам | Выручка и количество заказов на каждого менеджера |
| Статусы доставок | Сводка по статусам всех доставок |
| Итоги по поставщикам | Сумма и количество поставок по каждому поставщику |

### Отчёты по прибыли (с выбором периода)
Доход (выполненные заказы) vs расход (поставки) - по неделям, месяцам или годам.

### Параметрические запросы
| Запрос | Параметр |
|---|---|
| Товары по категории | Название категории |
| Заказы по датам | Диапазон дат |
| Товары по цене | Диапазон цен |
| Клиент по телефону | Номер телефона |
| Заказы клиента | ID клиента |
| Поиск товара | Часть наименования |

## Возможности интерфейса

Каждое окно таблицы поддерживает:
- просмотр всех записей
- добавление, редактирование, удаление
- живой поиск по всем колонкам
- сортировку по любой колонке (клик по заголовку)
- выпадающие списки для всех FK-полей

Вкладка **Заказы** открывает специальную форму с:
- выбором товаров и проверкой остатков на складе в реальном времени
- автоматическим подсчётом итоговой суммы

## Контейнерная проверка

Desktop GUI удобнее запускать локально на Windows, но контейнер фиксирует воспроизводимое Python-окружение для тестов:

Если используется структура из репозитория, команда из Makefile:

```bash
make compose-up
```

Напрямую:

```bash
docker compose -f infra/compose.yaml up --build
```

Контейнер запускает `python -m unittest discover -s tests`.
